View engines

  1. Introduction
  2. Registering new engines
  3. Existing engines

Introduction

The Opis Colibri framework has the ability to use multiple template engines simultaneously, thanks to the Opis View rendering engine. View engines are collected using the ViewEngines method. 

use Opis\Colibri\Define;

Define::ViewEngines(function($engine){

    // Collect engines

});

The collector passed to the callback function is an object that implements the Opis\Colibri\ViewEngineCollectorInterface interface. 

interface ViewEngineCollectorInterface
{
    /**
     * Defines a new view engine
     *
     * @param   \Closure    $constructor    A callback that will return an instance of \Opis\View\EngineInterface
     * @param   int         $priority       Engine's priority
     *
     * @return  \Opis\View\EngineEntry
     */
    
    public function register(Closure $constructor, $priority = 0);
}

Registering new engines

Engines can be registred by using the register method, on the collector instance, in conjunction with the handle method. 

use Opis\Colibri\Define;

Define::ViewEngines(function($engine){

    // Define an engine that handles all files that have a `.foo` extension

    $engine->register(function(){
        new FooEngine();
    })
    ->handle(function($path){
        return preg_match('/^.*\.foo/', $path);
    });

});

The register method receive as an argument a closure that must return an object that implements the Opis\View\EngineInterface interface.

The handle method receive as an argument another closure that must return a boolean value that indicates if the engine can handle the path passed as an argument to the closure.

Overriding engines

You can override an existing engine by passing a second parameter to the register method. This parameter must be an integer number and represents the engine's collecting priority. The default collecting priority of a engine is 0. 

use Opis\Colibri\Define;

Define::ViewEngines(function($engine){
    
    $engine->register(function(){
        new NewFooEngine();
    }, 1)
    ->handle(function($path){
        return preg_match('/^.*\.foo/', $path);
    });

});

Existing engines

By default, Opis Colibri has support only for the standard PHP template engine. You can use the popular Twig template engine inside your Opis Colibri project, by installing the twig module.

You can also take the twig module as a reference when building new view engines. The Twig integration is incredibly simple and the twig module gives the true measure of how easy is to integrate a template engine with Opis Colibri.
