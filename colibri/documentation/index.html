Installation

Before trying to install Opis Colibri make sure you have Composer installed on your machine. Opis Colibri utilizes Composer to manage its dependencies and it's preferable to have Composer globally available.

If you don't have composer installed, just type the following commands into your terminal in order to install it and make it globally available. 

curl -sS https://getcomposer.org/installer | php
sudo mv composer.phar /usr/local/bin/composer

Now you can install Opis Colibri by using Composer's create-project command. 

cd /var/www
composer create-project opis/colibri website.test

Apache configuration

Below is a minimum Apache configuration that you can use within your project. The configuration assumes that you've installed Opis Colibri into the /var/www folder, as showed above, and you have added website.test to your hosts file.

Important! Make sure that your Apache server has the mod_rewrite module enabled, otherwise Opis Colibri won't work. 

<VirtualHost *:80>
    
    ServerName  website.test
    DocumentRoot /var/www/website.test/public

    <Directory /var/www/website.test/public>
        AllowOverride All
        Order allow,deny
        allow from all
    </Directory>

</VirtualHost>

As you can see from the above configuration, your website's document root must be the framework's public folder. All other libraries, modules and files are kept outside the public root and cannot be directly accessed.

Inside the public folder you'll find the .htaccess file that ships with Opis Colibri. You can modify the .htaccess file to fit your needs.

Framework configuration

The Opis Colibri framework is configured during the installation process. Just open into a web browser your newly created site and follow the remaining installation steps.
