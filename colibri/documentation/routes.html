Introduction

The framework's routing system is based on Opis HTTP Routing library. Routes are collected using the Routes method. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    // Collect routes

});

You can override existing routes by specifying a higher collecting priority. 

use Opis\Colibri\Define;

// Override all routes that have a collecting priority lower than 1
Define::Routes(function($route){

    // Collect routes

}, 1);

The collector passed to the callback function is an object that implements the Opis\Colibri\RouteCollectorInterface interface. 

interface RouteCollectorInterface
{
    /**
     * Defines a global binding
     *
     * @param   string      $name       The name of the binding
     * @param   callable    $callback   A callback that will return the binding's value
     *
     * @return  \Opis\Colibri\RouteCollectorInterface   Self reference
     */
    
    public function bind($name, $callback);
    
    /**
     * Defines a global filter
     *
     * @param   string      $name       The name of the filter
     * @param   callable    $callback   A callback that will return the filter's value
     *
     * @return  \Opis\Colibri\RouteCollectorInterface   Self reference
     */
    
    public function filter($name, $callback);
    
    
    /**
     * Set a global wildcard
     *
     * @param   string  $name   The name of the wildcard
     * @param   string  $value  A regex expression
     *
     * @return  \Opis\Colibri\RouteCollectorInterface   Self reference
     */
    
    public function wildcard($name, $value);
    
    /**
     * Set a global implicit value for a wildcard
     *
     * @param   string  $name   The name of the wildcard
     * @param   mixed   $value  The implicit value
     *
     * @return  \Opis\Colibri\RouteCollectorInterface   Self reference
     */
    
    public function implicit($name, $value);
    
    /**
     * Defines a new route that will intercept all HTTP requests
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     * @param   string      $name   (optional) Route name
     *
     * @return  \Opis\HttpRouter\Route
     */
    
    public function all($path, $action, $name = null);
    
    /**
     * Defines a new route that will intercept all GET requests
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     * @param   string      $name   (optional) Route name
     *
     * @return  \Opis\HttpRouter\Route
     */
    
    public function get($path, $action, $name = null);
    
    /**
     * Defines a new route that will intercept all POST requests
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     *
     * @return  \Opis\HttpRouter\Route
     */
        
    public function post($path, $action, $name = null);
    
    /**
     * Defines a new route that will intercept all DELETE requests
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     * @param   string      $name   (optional) Route name
     *
     * @return  \Opis\HttpRouter\Route
     */
    
    public function delete($path, $action, $name = null);
    
    /**
     * Defines a new route that will intercept all PUT requests
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     * @param   string      $name   (optional) Route name
     *
     * @return  \Opis\HttpRouter\Route
     */
    
    public function put($path, $action, $name = null);
    
    /**
     * Defines a new route that will intercept all PATCH requests
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     * @param   string      $name   (optional) Route name
     *
     * @return  \Opis\HttpRouter\Route
     */
    
    public function patch($path, $action, $name = null);
    
    /**
     * Define a new route that will intercept  the specified methods
     *
     * @param   string|array    $path   The path to match
     * @param   callable        $action An action that will be executed
     * @param   string|array    $method (optional) Request method
     *
     * @return  \Opis\HttpRouter\Route
     */
    
    public function __invoke($path, $action, $method = null);
}

Creating routes

Routes are created by simply defining a path and a callable value that will act as a callback. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->get('/', function(){
        return 'Hello world';
    });
    
    $route->post('/', function(){
        return 'Hello world';
    });
    
    $route->put('/', function(){
        return 'Hello world';
    });
    
    $route->delete('/', function(){
        return 'Hello world';
    });
    
    $route->patch('/', function(){
        return 'Hello world';
    });

});

Another way of creating a route is through direct invocation of the collector object. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello world';
    });
    
    $route('/', function(){
        return 'Hello world';
    }, 'POST');
    
    $route('/', function(){
        return 'Hello world';
    }, 'PUT');
    
    $route('/', function(){
        return 'Hello world';
    }, 'DELETE');
    
    $route('/', function(){
        return 'Hello world';
    }, 'PATCH');

});
Respond to any HTTP verb

Creating a route that responds to any HTTP verb is done by using the all method. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->all('/', function(){
        return 'Hello World';
    });

});
Respond to multiple HTTP verbs

You can also create a route that responds to a list of HTTP verbs. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->all('/', function(){
        return 'Hello World';
    })
    ->method(['GET', 'POST']);
    
    // OR ..
    
    $route('/', function(){
        return 'Hello world';
    }, ['GET', 'POST']);

});
Named routes

Named routes can be used to generate a new URL from a specific route with the help of the PathFromRoute global function. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->get('/', function(){
        return 'Hello World';
    }, 'name');
    
    // OR ..
    
    $route(['name' => '/'], function(){
        return 'Hello world';
    });

});

Route parameters

Parameters are used to capture segments of the route's path and they are always enclosed by curly braces. A parameter can be used inside the route's callback function by adding the parameter's name to the argument list of the callback function. Therefore a parameter's name must not contain characters that are not allowed to be used for naming PHP variables. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/message/{id}', function($id){
        //
    });

});

You can define as many parameters as you need. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/inbox/{mailbox}/read/{id}', function($mailbox, $id){
        //
    });

});
Optional parameters

Specifying that a parameter is optional can be done by adding a ? mark after the parameter name. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/profile/{id?}', function($id = 0){
        //
    });

});
Implicit values

You can add an implicit value for a parameter name, using the implicit method on a route instance. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/profile/{id?}', function($id){
        //
    })
    ->implicit('id', 0);

});

An implicit value for a parameter name can also be made globally available by calling the implicit method on the collector instance. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->implicit('id', 0);

    $route('/profile/{id?}', function($id){
        // Returns $id or 0
        return $id;
    });
    
    // Overwrite
    $route('/user/{id?}', function($id){
        // Retruns $id or 1
        return $id;
    })
    ->implicit('id', 1);

});
Regex constraints

You can add regex constraints to parameters by using the where method. The method takes as arguments the parameter name and a regular expression defining how the parameter should be constrained. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/profile/{id}', function($id){
        //
    })
    ->where('id', '[0-9]+');
    
    $route('/user/{name}', function($name){
        //
    })
    ->where('name', '[a-zA-Z]+');
    
    $route('/inbox/{mailbox}/{id}', function($id, $mailbox){
        //
    })
    ->where('id', '[0-9]+')
    ->where('mailbox', '[a-zA-Z]+');

});

You can make a regex constraint globally available by using the wildcard method. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->wildcard('id', '[0-9]+');

    $route('/profile/{id}', function($id){
        // $id use global constraint [0-9]+ 
    });
    
    //Overwrite
    $route('/user/{id}', function($id){
        // $id use local constraint [a-zA-Z]+
    })
    ->where('id', '[a-zA-Z]+');

});

Special implicit values

There are two special implicit values: self and path. The self implicit's value is the current route instance and path implicit's value is the current path object, which is an instance of the Opis\HttpRouting\Path class. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function($self, $path){
        //
    });

});

Domain constraints

Some routes can be constrained to be available only for a specific domain or subdomain. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello World!';
    })
    ->domain('mysite.com');
    
});

You can also use parameters when defining a domain for a route. The parameters can be used inside the route's callback function and you can apply regex constraints to domain parameters. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/echo', function($subdomain){
        return $subdomain;
    })
    ->domain('{subdomain}.mysite.com')
    ->where('subdomain',  'blog|forum|www');
    
});

Domain parameters can also be marked as being optional. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/echo', function($subdomain = 'www'){
        return $subdomain;
    })
    ->domain('{subdomain?}.mysite.com')
    ->where('subdomain',  'blog|forum|www');
    
});

Secure connection constraint

Routes can be constrained to be available only when they are accessed using secure HTTP connections. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/payment', function(){
        //
    })
    ->secure();
    
});

Bindings

Binding a value to an identifier is done by using the bind method on a route instance. The method takes as arguments the identifier and a callback. The newly created identifier can then be referenced by route's callback. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function($value){
        return $value;
    })
    ->bind('value', function(){
        return 'Hello world!';
    });

});

Another usage for the bind method is to overwrite a parameter's value 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/echo/{text}', function($text){
        return $text;
    })
    ->bind('text', function($text){
        return strtoupper($text);
    });
    
});

An identifier can also be bound to a value that was formed by using multiple parameters. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/entity/{type}/{id}', function($entity){
        // Do something with the entity object
    })
    ->bind('entity', function($type, $id){
        return Database()->from('entities')
                         ->where('type')->is($type)
                         ->andWhere('id')->is($id)
                         ->select()
                         ->first();
    });
    
});
Global bindings

Bindings can be made globally available by using the bind method on the collector instance. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->bind('entity', function($type, $id){
        return Database()->from('entities')
                         ->where('type')->is($type)
                         ->andWhere('id')->is($id)
                         ->select()
                         ->first();
    });

    $route('/entity/{type}/{id}', function($entity){
        // 
    });
    
    /* Use a parameter and an implicit value to create the entity */
    $route('/user/{id}', function($entity){
        //
    })
    ->implicit('type', 'user');
    
});

Filters

Creating a local filter is done by using the filter method on a route instance. The filter method takes as arguments the name of the filter and a valid callable value that will act as a callback. Depending if the route passes the filter or not, the callback must return either true or false. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello world';
    })
    ->filter('foo', function(){
        return true;
    });
    
});
Global filters

You can also create a global filter by calling the filter method on the collector instance. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route->filter('foo', function(){
        return true;
    });
    
});
Using filters

Simply creating a local or a global filter has no effect over a route and it does not influence the routing process. Using a filter with a route is done by using one of the following methods on a route instance: before, after and access.

Before

Filters added using the before method will be executed before the binding process takes place. Filters are executed only if the path and the domain pass all defined constraints. If a route does not pass a before filter, the route is ignored and the next route in the list gets checked. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello world';
    })
    ->filter('foo', function(){
        return true;
    })
    ->before('foo');
    
});

You can add multiple filters by using, as an argument to the method, an array of filter names. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello world';
    })
    ->before(['foo', 'bar', 'baz']);
    
});
After

Filters added using the after method gets executed only if all the before filter passes. At the point when an after filter is executed, all the bindings have been finalized. If a route does not pass an after filter, then a 404 Not found error page is returned. You can add multiple filters by using, as an argument to the method, an array of filter names. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello world';
    })
    ->before(['qux', 'baz'])
    ->after('foo');
    
    $route('/path', function(){
        return 'Hello world';
    })
    ->after(['foo', 'bar']);
    
});
Access

Filters added using the access method gets executed only if all the after filter passes. If a route does not pass an access filter, then a 403 Forbidden error page is returned. You can add multiple filters by using, as an argument to the method, an array of filter names. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function(){
        return 'Hello world';
    })
    ->access('foo');
    
    $route('/path', function(){
        return 'Hello world';
    })
    ->access(['foo', 'bar']);
    
});

Stored values

Storing an arbitrary value into a route instance using a key named foo is done by calling a method foo on the route instance. 

use Opis\Colibri\Define;

Define::Routes(function($route){

    $route('/', function($self){
        $foo = $self->get('foo');
        $bar = $self->get('bar');
        $baz = $self->get('baz');
    })
    ->foo('fooValue')
    ->bar(array(1,2,3))
    ->baz(function(){
    
    });
    
});
