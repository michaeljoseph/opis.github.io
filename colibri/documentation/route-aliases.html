Route aliases

  1. Introduction
  2. Creating route aliases
  3. Advanced concepts

Introduction

Route aliases are used to provide aliases for already registered routes. For example a not so SEO friendly path like /blog/article/123 could be aliased as /the-title-of-my-article-123.html, which is much more SEO friendly. The framework's aliasing system is based on Opis Routing library. Route aliases are collected using the RouteAliases method. 

use Opis\Colibri\Define;

Define::RouteAliases(function($route){

    // Collect route aliases

});

You can override existing route aliases by specifying a higher collecting priority. 

use Opis\Colibri\Define;

// Override all route aliases that have a collecting priority lower than 1
Define::RouteAliases(function($route){

    // Collect route aliases

}, 1);

The collector passed to the callback function is an object that implements the Opis\Colibri\RouteAliasCollectorInterface interface. 

interface RouteAliasCollectorInterface
{
    /**
     * Defines an alias for a route or a group of routes
     *
     * @param   string      $path   The path to match
     * @param   callable    $value  An action that will be executed
     *
     * @return  \Opis\Routing\Route
     */
    
    public function alias($path, $action);
}

Creating route aliases

Routes aliases are created by passing a path and a callback value as arguments to the alias method. The callback must return a string representing the aliased path. 

use Opis\Colibri\Define;

Define::RouteAliases(function($route){

    $route->alias('/some-alias-of-a-path', function(){
        return '/real-path';
    });

});

Advanced concepts

Since both the routing system and the aliasing system have on their very base the Opis Routing library, they have a lot of common concepts.

You can specify parameters and optional parameters for your alias route. 

use Opis\Colibri\Define;

Define::RouteAliases(function($route){

    $route->alias('/article/{title}', function($title){
        //
    });
    
    $route->alias('/user/{name?}', function($name = 'me'){
        //
    });

});

You can provide implicit values for parameters. 

use Opis\Colibri\Define;

Define::RouteAliases(function($route){
    
    $route->alias('/user/{name?}', function($name){
        //
    })
    ->implicit('name', 'me');

});

You can define regex constraints for parameters by using the wildcard method. 

use Opis\Colibri\Define;

Define::RouteAliases(function($route){
    
    $route->alias('/user/{name?}', function($name){
        //
    })
    ->wildcard('name', '[a-zA-Z][a-zA-Z0-9_]+')
    ->implicit('name', 'me');

});

You can also bind a value to an identifier. 

use Opis\Colibri\Define;

Define::RouteAliases(function($route){
    
    $route->alias('/user/{name?}', function($name){
        //
    })
    ->wildcard('name', '[a-zA-Z][a-zA-Z0-9_]+')
    ->implicit('name', 'me')
    ->bind('name', function($name){
        return strtolower($name);
    });

});
