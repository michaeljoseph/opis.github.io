Dispatchers

  1. Introduction
  2. Registering dispatchers
  3. Creating dispatchers
  4. Using dispatchers

Introduction

Dispatchers are a very important component of the framework's routing system and they are used to dispatch requests to routes. Users can define their own dispatchers that will handle the routes in a way that fit their needs. Dispatchers are collected using the Dispatchers method. 

use Opis\Colibri\Define;

Define::Dispatchers(function($dispatcher){

    // Collect dispatchers

});

You can override registred dispatchers by specifying a lower collecting priority. 

use Opis\Colibri\Define;

// Override all dispatchers that have a collecting priority greater than -1
Define::Dispatchers(function($dispatcher){

    // Collect dispatchers

}, -1);

The collector passed to the callback function is an object that implements the Opis\Colibri\DispatcherCollectorInterface interface. 

interface DispatcherCollectorInterface
{
    /**
     * Register a new dispatcher
     *
     * @param   string      $name       Dispatcher's name
     * @param   callable    $builder    A callback that will return a dispatcher
     *
     * @return  \Opis\Colibri\DispatcherCollectorInterface  Self reference
     */
    
    public function register($name, $builder);
}

Registering dispatchers

Registering a dispatcher is done by passing as arguments to the register method the name of the dispatcher and a callable value that will act as a callback. The callback must return an object that inherits from Opis\Routing\Dispatcher class. 

use Opis\Colibri\Define;
use Opis\Colibri\Module\Foo\Dispatcher;

Define::Dispatchers(function($dispatcher){

    $dispatcher->register('foo', function(){
        return new Dispatcher();
    });

});

Creating dispatchers

In order to create a new dispatcher you must create a new class that inherits from Opis\Routing\Dispatcher or one of its subclasses. The best way to start when creating a new dispatcher is to study the JSON dispatcher provided by the system module. 

namespace Opis\Colibri\Module\Foo;

use Opis\Routing\Path;
use Opis\Routing\Route;
use Opis\Routing\Dispatcher as BaseDispatcher;

class Dispatcher extends BaseDispatcher
{
    public function dispatch(Path $path, Route $route)
    {
        if($route->has('foo'))
        {
            return $route->get('foo');
        }
        
        return parent::dispatch($path, $route);
    }
}

Using dispatchers

You can tell a route which dispatcher to use by passing the dispatcher's name to the route's dispatcher method. 

use Opis\Colibri\Define;
use Opis\Colibri\Module\Foo\Dispatcher;

Define::Dispatchers(function($dispatcher){

    $dispatcher->register('foo', function(){
        return new Dispatcher();
    });

});

Define::Routes(function($route){

    $route('/path', function(){
        //
    })
    ->dispatcher('foo');

});
