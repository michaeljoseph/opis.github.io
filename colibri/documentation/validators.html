Introduction

Validating user submitted content is one of the most challenging and frequently used operation in web applications. Opis Colibri aims to make form validation process much more simple and more secure, by providing a set of predefined validators along with the possibility to easily define new validators. Validators are collected using the Validators method. 

use Opis\Colibri\Define;

Define::Validators(function($validators){

    // Collect validators

});

You can override a validators by specifying a lower collecting priority. 

use Opis\Colibri\Define;

Define::Validators(function($validators){

    // Collect validators

}, -1);

The collector passed to the callback function is an object that implements the Opis\Colibri\ValidatorCollectorInterface interface. 

interface ValidatorCollectorInterface
{
    /**
     * Register a new validator
     *
     * @param   string      $name       Validator's name
     * @param   callable    $callback   Validator's implementation
     *
     * @return  \Opis\Colibri\ValidatorCollectorInterface   Self reference
     */
    
    public function register($name, $callback);
}

Registering new validators

You can register a new validator by passing the validator's name and callback to the register method. The first argument of the callback will be an instance of the Opis\Colibri\Validator class. The callback must return an array that contains a validator key and a error key.

The value contained on the validator key must be an array that contains a validation callback and an array of arguments that will be passed to the validation callback. The value that needs to be validated is inserted at the begining of the arguments list every time the validator is invoked. The validator callback must return a boolean value.

The value contained on the error key must be an array that contains the error text and a list of variables that will replace every occurrence of %s in the error text, in the order on which they appear. You can get the field's name being validated by using the getFiled method on the validator instance. Note that you must get the field's name through reference. 

use Opis\Colibri\Define;

Define::Validators(function($validators){

    $validators->register('alphanum', function($validator, $error = '%s must be alphanumeric'){
        
        return array(
            'validator' => array(
                // validator
                'callback' => function($value){
                    return preg_match('/[a-zA-Z0-9]+/', $value);
                },
                // $value will be inserted at the begining
                'arguments' => array(), 
            ),
            'error' => array(
                // error text
                'text' => $error, 
                // replace first %s with field's name
                'variables' => array(&$validator->getField()), 
            ),
        );
        
    });

});

How to use validators

Validators can be used by simply creating a new instance of Opis\Colibri\Validator. You can use a validator named foo by invoking the foo method on the validator instance. The field's name and the field's value that need to be validated are passed to the validate method. The validate method returns the field's value passed as an argument. 

use Opis\Colibri\Validator

$validator = new Validator();

$fieldValue = $validator->foo()
                        ->validate('Field', $value);

Of course you can use more than one validator to validate a value and you can validate multiple values with the same instance of Opis\Colibri\Validator class. 

use Opis\Colibri\Validator

$request = HttpRequest();
$validator = new Validator();

$username = $validator->required()
                      ->minLength(3)
                      ->maxLenght(32)
                      ->alphanum()
                      ->validate('Username', trim($request->post('username')));
                      
$password = $validator->required()
                      ->minLength(6)
                      ->maxLength(12)
                      ->validate('Password', trim($request->post('password')));

Checking if there are any validation errors is done using the hasErrors method. You can get the list of errors using the getErrors method. 

if($validator->hasErrors())
{
    $errors = $validator->getErrors();
    
    foreach($errors as $error)
    {
        //do something
    }
}

If the system module is enabled, you can also use the toSystemAlerts method. 

if($validator->hasErrors())
{
    $validator->toSystemAlerts();
    return;
}

// No validation erros

Predefined validators

The framework provides a number of predefined validators that cannot be replaced by the collected validators. Below is a list of predefined validators that can be used when validating user submitted content.

required

Check if a field's value is not empty or null.
Arguments:

  • $error - (optional) Error text (%s is required) 
$validator->required()
          ->validate('Field', $value);
equalLength

Check if a field's value has the required length.
Arguments:

  • $lenght - Required length
  • $error - (optional) Error text (%s must have precisely %s character(s) long) 
$validator->equalLenght(14)
          ->validate('Field', $value);
minLength

Check if a field's value has the required minimum length.
Arguments:

  • $lenght - The required minimum length
  • $error - (optional) Error text (%s must be at least %s character(s) long) 
$validator->minLength(3)
          ->validate('Field', $value);
maxLength

Check if a field's value has the required maximum length.
Arguments:

  • $lenght - The required maximum length
  • $error - (optional) Error text (%s must be at most %s character(s) long) 
$validator->maxLength(32)
          ->validate('Field', $value);
equal

Check if a field's value is equal to a given value.
Arguments:

  • $value - The value to compare with
  • $error - (optional) Error text (%s must be equal with %s) 
$validator->equal('some value')
          ->validate('Field', $value);
gt

Check if a field's value is greater than a given value.
Arguments:

  • $value - The value to compare with
  • $error - (optional) Error text (%s must be greater than %s) 
$validator->gt(3)
          ->validate('Field', $value);
gte

Check if a field's value is equal or greater than a given value.
Arguments:

  • $value - The value to compare with
  • $error - (optional) Error text (%s must be at least %s) 
$validator->gte(3)
          ->validate('Field', $value);
lt

Check if a field's value is smaller than a given value.
Arguments:

  • $value - The value to compare with
  • $error - (optional) Error text (%s must be less than %s) 
$validator->lt(32)
          ->validate('Field', $value);
lte

Check if a field's value is equal or smaller than a given value.
Arguments:

  • $value - The value to compare with
  • $error - (optional) Error text (%s must be at most %s) 
$validator->lte(32)
          ->validate('Field', $value);
between

Check if a field's value falls within a given range.
Arguments:

  • $min - The range's minimum value
  • $max - The range's maximum value
  • $error - (optional) Error text (%s must be between %s and %s) 
$validator->between(18, 99)
          ->validate('Field', $value);
match

Check if a field's value match another field.
Arguments:

  • $value - Other's field value
  • $field - Other's field name
  • $error - (optional) Error text (%s must match %s) 
$value1 = $validator->required()
                    ->validate('Field1', trim($fieldValue1));
                    
$value2 = $validator->match($value1, 'Field1')
                    ->validate('Field2', trim($fieldValue2));
email

Check if a field's value is an email address.
Arguments:

  • $error - (optional) Error text (%s must be a valid email) 
$validator->email()
          ->validate('Field', $value);
number

Check if a field's value is a number.
Arguments:

  • $error - (optional) Error text (%s must be a number) 
$validator->number()
          ->validate('Field', $value);
regex

Check if a field's value match a regex pattern.
Arguments:

  • $pattern - Pattern to match
  • $error - (optional) Error text (%s is not valid) 
$validator->regex('/[a-z0-9]+/')
          ->validate('Field', $value);
fileRequired

Check if a field's value is an uploaded file.
Arguments:

  • $error - (optional) Error text ('%s is required') 
$validator->fileRequired()
          ->validate('Field', $value);
fileType

Check if a field's value is a file of the specified MIME type.
Arguments:

  • $type - MIME type
  • $error - (optional) Error text ('Invalid file type') 
$validator->fileRequired()
          ->fileType('image/png')
          ->validate('Field', $value);
fileMatch

Check if a field's value is file and the file's name match a given regex pattern.
Arguments:

  • $pattern - Pattern to match
  • $error - (optional) Error text (Invalid file type) 
$validator->fileRequired()
          ->fileType('image/png')
          ->fileMatch('/^.*\.png$/')
          ->validate('Field', $value);
csrf

Check if a field's value is a valid CSRF token.
Arguments:

  • $error - (optional) Error text (Invalid CSRF token) 
$validator->csrf()
          ->validate('Field', $value);
