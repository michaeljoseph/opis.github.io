Modular framework

Opis Colibri follows a modular architecture and treats each module as a standalone component that can be individually disabled or enabled on demand. Modules are reusable across all Opis Colibri based projects, offering a great way of extending existing web applications and maintaining large code bases clean and well structured.

Decoupled components

In contrast to other frameworks, Opis Colibri is builded from standalone libraries that can be used with any modern PHP framework. Each and every library was built to serve its own purpose and it's fully decoupled from the Opis Colibri framework, meaning that any feature of Opis Colibri can be ported to other systems.

Open source

Opis Colibri is an open source project licensed under Apache 2.0 license. The source code of this project is publicly available on GitHub and everyone is free to use, to modify, distribute or sublicense Opis Colibri or its standalone components, for both commercial or non-commercial purposes.

Composer compatible

Composer is a great tool for managing project dependencies and is fully compatible with Opis Colibri. Use Composer to enhance your web application functionalities by using third-party libraries listed on Packagist.

Great routing system

Opis Colibri provides developers a high degree of freedom and flexibility when it comes to define complex routing patterns. Developers can control in slight details how a specific route will be handled by the framework.

Flexible templating

When dealing with templates, Opis Colibri takes an unique approach. It allows you to use multiple rendering engines simultaneously and does not impose you how to store and retrieve your templates.

Developer-friendly

Opis Colibri follows an OOP paradigm and intensively uses anonymous functions to define callbacks. Principles as "code reusability" or "separation of concerns" are highly valued, making Opis Colibri the perfect tool for building new web applications.
